アップロードしていただいたExcelの計算シート（csvデータ）をベースに、FGOにおける「絆ポイントの計算方法と仕組み」について分かりやすく解説します！

このシートは、クエストで獲得できる基礎絆ポイントに対して、各種ボーナス（スタメンボーナスや絆礼装など）がどのように掛け合わされて最終的な獲得絆ポイント（あるいはTP効率など）になるかを非常に論理的に整理されていますね。

以下に、シートの構成に沿って計算の流れを詳しく解説していきます。

---

### 📋 絆計算の基本フロー（シートの仕組み）

FGOの絆計算は、大きく分けて以下の**2つの段階**を踏んで計算されます。

1. **第1段階：クラスボーナス・スタメンボーナスの適用（C = A * B）**
2. **第2段階：特攻イベント・絆礼装（20%礼装など）の適用（Y = C * X、合計 = C + Y）**

シートのアルファベット（A〜Y）を基に、具体的に見ていきましょう。

---

### 🔍 各項目の意味と計算の詳細

#### 1. 基礎値とスタメン判定（第1段階）

* **A：編成（基礎絆ポイント）**
* クエストごとに設定されている、基本の絆ポイントです（例：シートでは `4748` となっています）。


* **B：スタメン（クラス・配置ボーナス倍率）**
* 特定のイベントや新章の「クラス限定ボーナス」や「スタメン配置ボーナス」の倍率です。
* シートの1〜2番目の鯖は `1.24`倍（24%アップ）、3〜5番目の鯖は `1.04`倍（4%アップ）となっており、配置やクラスによる差が忠実に再現されています。


* **C：1段階絆（ `A * B` ）**
* 基礎ポイントにスタメンボーナスを掛け合わせた、**中間の基準値**になります（例：`4748 * 1.24 = 5887`）。
* **重要：** 後述する「絆礼装（%アップ）」は、最初のAではなく、**この1段階絆（C）をベースに計算**されます。



#### 2. 礼装・キャンペーンの加算（第2段階）

* **D：イベ・キャンペーン**
* イベント特攻などの倍率です（シートでは今回 `0` になっています）。


* **X：絆礼装合計**
* 今回まとめようとしている「絆20%アップ礼装（0.2）」や、カルデア・ランチタイム（10%＝0.1）、ティータイム（マロニエ含むスタメン15%＝0.15）などの合計値です。
* シートの1番目の鯖は `0.65`（計65%アップ：例えば15% + 10% + 20% + 20% など）、2番目の鯖は `1.05`（計105%アップ）のように、それぞれの鯖が持つ**特性の該当数**によって倍率が綺麗に変動しています。


* **Y：増加絆（ `C * X` ）**
* 1段階絆（C）に対して、礼装の合計倍率（X）を掛けた「ボーナス分のみのポイント」です。



#### 3. 最終結果

* **合計値（ `C + Y` ）**
* 「1段階の絆ポイント」に「礼装での増加分」を足した、**そのサーヴァントが1周で実際に獲得できる最終的な絆ポイント**です。
* シートの例1：`5887（C） + 3826（Y） = 9713`


* **TP / 結果(TP)**
* 消費APに対する効率や、特定の周回数（あるいは特定のタイム・ポイント換算）を踏まえた全体の獲得効率を表しています。全員の合計値を出すことで、1周（または1編成）でどれだけの総絆ポイントを稼げているかが一目でわかるようになっています。

### 1. 絆20%アップ礼装の基本仕様

現在FGOで実装されている特定の絆アップ礼装（「NFFから愛をこめて」「カルデア・モーニング」「レポートチェック」など）は、従来の「一律5%〜15%アップ」の礼装とは異なり、**特定のクラス・属性・隠し特性を持つサーヴァント（鯖）に対してのみ【20%（0.20）】のボーナスを付与する**という特徴を持っています。

### 2. 管理すべき10個の対象特性（フィルター項目）

効率よく20%ボーナスを重複（累積）させるために、以下の10項目に該当するかどうかが鍵となります。

1. **キャスター**（クラス）
2. **ライダー**（クラス）
3. **ケモノ科**（特性）
4. **今を生きる人類**（特性）
5. **秩序かつ善**（属性：秩序 ＆ 善）
6. **霊衣を持つ者**（仕様：霊衣が実装されている鯖）
7. **秩序かつ女性**（属性：秩序 ＆ 性別：女性）
8. **星または悪**（属性：星 ＆ または属性：悪）
9. **混沌かつ七騎士**（属性：混沌 ＆ クラス：剣/弓/槍/騎/術/殺/狂のいずれか）
10. **中立**（属性：中立・善 / 中立・中庸 / 中立・悪などの「中立」持ち）

### 3. 絆計算の仕組み（計算シートのロジック）

アップロードいただいた計算シートの通り、絆ポイントの計算は「掛け算」と「足し算」の2段階で行われます。20%礼装の倍率は第2段階の加算（X）に影響します。

* **第1段階：1段階絆（C）の算出**

$$\text{基礎絆ポイント（A）} \times \text{スタメン・クラス配置ボーナス（B）} = \text{1段階絆（C）}$$



*(例：$4748 \times 1.24 = 5887$)*
* **第2段階：増加絆（Y）の算出と最終合計**
20%礼装を含む「絆礼装合計（X）」を、基礎値（A）ではなく1段階絆（C）に掛け合わせます。

$$\text{1段階絆（C）} \times \text{絆礼装合計（X）} = \text{増加絆（Y）}$$


$$\text{最終獲得絆} = \text{1段階絆（C）} + \text{増加絆（Y）}$$



*(例：礼装合計が105%（1.05）の場合、 $5887 \times 1.05 = 6181$ となり、最終合計は $5887 + 6181 = 12068$)*

### 4. 運用上の重要な注意点（再臨・霊衣による変化）

AppMediaのツールにも記載されている通り、20%礼装の対象判定には一部のサーヴァントで「再臨段階」や「霊衣の着脱」によって特性が変化し、対象外（または対象内）になるケースがあります。

* **再臨段階による変化の例：**
* `水着キアラ`、`水着クロエ`、`BBドバイ`、`ロウヒ` などは【第三再臨のみ】特定の特性を満たす、あるいは変化する。
* `トネリコ（水着モルガン）` は【第一、第二再臨のみ】該当。
* `U-オルガマリー` は再臨段階によって細かく対象礼装が変わる。


* **霊衣による変化の例：**
* `両儀式（剣）`、`スカサハ`、`バニヤン` などは【霊衣を着用している時のみ】「霊衣を持つ者（至誠の一針）」の20%対象に該当する。
